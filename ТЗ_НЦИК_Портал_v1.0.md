### 1.1 Основание для разработки

Разработка веб-портала проводится в соответствии с:

- Постановлением Президента Республики Узбекистан от 19 ноября 2025 г. об организации деятельности Национального комитета по экологии и изменению климата, которым предусмотрено создание единой экологической онлайн-платформы не позднее 1 сентября 2026 г.;
- необходимостью обеспечить прозрачность деятельности Центра, публичный доступ к аналитическим материалам и официальным документам;
- потребностью в профессиональном многоязычном инструменте коммуникации с национальными и международными партнёрами.

### 1.2 Срок выполнения работ

Ориентировочный срок разработки — **4 месяца** с момента подписания договора (см. раздел 17).

## 3. Назначение и цели портала

### 3.1 Назначение

Портал является официальным публичным представительством НЦИК в сети Интернет. Портал обеспечивает:

- информирование широкой аудитории о деятельности Центра, климатической политике и нормативно-правовой базе;
- публикацию аналитических материалов, докладов и официальных документов;
- оперативное освещение новостей и решений в области изменения климата;
- взаимодействие с партнёрами, СМИ и гражданами.

### 3.2 Ключевые цели

| № | Цель | Измеримый результат |
|---|---|---|
| 1 | Обеспечить прозрачность деятельности НЦИК | Все официальные документы, решения и отчёты размещены на сайте |
| 2 | Стать авторитетным источником по климатике для Узбекистана | Позиции в ТОП-3 поисковой выдачи по профильным запросам |
| 3 | Упростить управление контентом для персонала Центра | Время публикации материала — не более 5 минут без технических знаний |
| 4 | Соответствовать международным стандартам государственных порталов | WCAG 2.1 уровень AA, HTTPS, поддержка 3 языков |
| 5 | Автоматизировать публикацию данных мониторинга | Интеграция с внешними API (качество воздуха, погода, Всемирный банк) |

### 6.2 Описание ролей

#### Администратор

- Полный доступ ко всем разделам CMS.
- Создаёт учётные записи модераторов и управляет их профилями.
- Управляет структурой меню, настройками сайта, SEO-метаданными.
- Видит журнал всех действий пользователей.
- Может восстанавливать удалённый контент из архива (срок хранения: 30 дней).
- Получает уведомления о публикациях модераторов (e-mail / push).

#### Модератор

- Создаёт и редактирует новости, страницы, публикации, документы и решения.
- Не может удалять материалы, созданные другими пользователями.
- Не может изменять структуру меню и системные настройки.
- Работает только с контентом в рамках назначенных разделов (принцип наименьших полномочий).
- Видит свою историю действий за последние 90 дней.

#### Гость (посетитель)

- Только просмотр опубликованного контента.
- Нет доступа к панели управления.
- Не требует регистрации для чтения материалов.

### 6.3 Аутентификация и сессии

- Вход по логину + паролю (парольные хэши bcrypt, не менее 12 итераций).
- Поддержка двухфакторной аутентификации (2FA) по TOTP (Google Authenticator, Authy) — обязательна для администраторов.
- Срок сессии: 8 часов при активности, 2 часа при бездействии.
- Автоматический выход при закрытии браузера (optional флаг «Запомнить»).
- Блокировка учётной записи после 5 неудачных попыток входа (разблокировка администратором).
- Отдельная страница восстановления пароля через e-mail.

## 7. Функциональные требования — публичная часть

### 7.1 Главная страница

**Блоки:**

1. **Шапка (Header)**
   - Логотип НЦИК + полное наименование организации.
   - Основное меню навигации (горизонтальное, адаптивное).
   - Переключатель языков: O'z | Рус | En.
   - Переключатель темы: светлая / тёмная.
   - Кнопка поиска.
   - Мобильное меню (гамбургер).

2. **Герой-блок (Hero)**
   - Заголовок: название центра на активном языке.
   - Краткая миссия.
   - Статус-строка (дата последнего обновления, количество опубликованных материалов).

3. **Основные направления**
   - 3 карточки ключевых направлений деятельности с иконками, заголовком, кратким описанием и ссылкой.

4. **Лента последних новостей** (горизонтальный прокручиваемый блок)
   - До 10 последних новостей в виде карточек с датой, тегом, заголовком и ссылкой.

5. **Последние решения/документы**
   - Список из 5 последних решений или постановлений со ссылками.

6. **Блок NDC**
   - Краткие целевые показатели национально определённых вкладов (визуализация в виде карточек).

7. **Качество воздуха и погода**
   - Индекс качества воздуха AQI в Ташкенте (реальное время).
   - Текущая погода (температура, условия).

8. **Блок Green Cities Rating**
   - Краткое описание + ссылка на платформу greencities.uz.

9. **Подвал (Footer)**
   - Контактная информация.
   - Быстрые ссылки.
   - Навигация по разделам.
   - Копирайт и политика конфиденциальности.

### 7.2 Раздел «О центре»

- Описание организации (многоязычный текст, редактируется из CMS).
- Блок «Руководство и команда»: карточки сотрудников с фото (портрет), именем, должностью, кратким описанием.
- Хронология / история деятельности (опционально).
- Фотогалерея (здание, мероприятия).
- Нормативная база (список документов о деятельности НЦИК).

### 7.3 Раздел «Деятельность» — NDC

- Представление трёх версий НОВ (NDC 1.0, 2.0, 3.0) в виде карточек с годами и ключевыми показателями.
- Визуализация целевых значений (снижение ПГ, ВИЭ, энергоэффективность) в виде цифровых плиток.
- Секторальная разбивка (вода, сельское хозяйство, здравоохранение, энергетика, лесное хозяйство, отходы).
- Список ключевых документов с кнопкой «Открыть документ» (внешние ссылки UNFCCC).

### 7.4 Раздел «Новости»

- Список новостей (карточки): дата, тег (цветной бейдж), заголовок, краткое описание, фото (опционально), ссылка «Читать далее».
- Строка поиска по заголовку и тексту.
- Фильтрация по тегу.
- Пагинация или бесконечная загрузка (lazy load).
- Страница отдельной новости: заголовок, дата, автор (опционально), тело публикации с поддержкой форматирования, изображения, ссылки на источник, кнопка «Назад».

### 7.5 Раздел «Документы / Публикации»

- Разбивка по типам: Решения, Публикации, Официальные документы, Доклады.
- Каждый документ: наименование, год, тип, краткое описание, кнопка «Скачать / Открыть».
- Фильтрация по году и типу.
- Поддержка форматов: PDF, DOCX, XLSX, PPTX.

### 7.6 Раздел «Данные и аналитика»

- Таблицы и графики экологических показателей Узбекистана (данные Всемирного банка): CO₂ на душу населения, ВИЭ, лесной покров, PM2.5.
- Список источников с названием, организацией-источником и кликабельной ссылкой.

### 7.7 Раздел «Контакты»

- Адрес, телефон, e-mail.
- Карта (OpenStreetMap embed — без Google Maps для соответствия политике конфиденциальности).
- Форма обратной связи (имя, e-mail, тема, сообщение, CAPTCHA).
  - Сообщения поступают на почту редакции и сохраняются в CMS.
- График работы.

### 7.8 Поиск по сайту

- Полнотекстовый поиск по заголовкам и телам новостей, страниц и документов.
- Постраничные результаты с подсветкой совпадений.
- Поиск независим от языкового переключателя (ищет по текущему языку).

## 8. Функциональные требования — панель управления

### 8.1 Общие требования к CMS

- Отдельный защищённый URL (`/admin/`), не индексируемый поисковиками.
- Адаптивный интерфейс (работа как с ноутбука, так и с планшета).
- Язык интерфейса CMS: русский (переключатель: рус/узб/eng — опционально).

### 8.2 Вход и сессия

- Форма входа: логин + пароль + 2FA (для администраторов).
- Автовыход при бездействии (настраивается: 30–480 минут).
- Уведомление о параллельном сеансе.

### 8.3 Главная панель (Dashboard)

- Счётчики: новостей, документов, страниц, необработанных сообщений от посетителей.
- График публикаций за последние 30 дней.
- Последние действия (журнал активности).
- Уведомления и задачи.

### 8.4 Управление новостями

## 9. Требования к дизайну и UX

### 9.1 Общий стиль

- **Характер:** профессиональный, государственный, современный, экологический.
- **Цветовая палитра:** оттенки зелёного и сине-зелёного (teal) как основные цвета, белый/серый как базовые, умеренные акценты.
- **Шрифты:** системные или Google Fonts (без лицензионных ограничений); поддержка кириллицы обязательна.
- **Иконки:** SVG-иконки (линейные), нейтральные и профессиональные.

### 9.2 Тёмная тема

- Поддержка тёмной темы оформления (Dark Mode).
- Автоопределение предпочтения системы (`prefers-color-scheme`).
- Ручной переключатель — сохраняется в `localStorage`.
- Все цвета, контрасты и изображения корректно адаптированы для тёмной темы.

### 9.3 Адаптивность (Responsive Design)

| Брейпойнт | Ширина | Поведение |
|---|---|---|
| Mobile | < 480 px | Стопка, одна колонка, гамбургер-меню |
| Tablet | 480–860 px | Стопка / 2 колонки, гамбургер-меню |
| Desktop | > 860 px | Полный горизонтальный layout |

- Сайт должен корректно отображаться и управляться на мобильных устройствах.
- Touch-friendly: кнопки не менее 44×44 px.

### 9.4 Доступность (Accessibility)

- Соответствие стандарту WCAG 2.1 уровня AA:
  - Контраст текста: не менее 4.5:1 (обычный), 3:1 (крупный).
  - Все интерактивные элементы доступны с клавиатуры.
  - ARIA-атрибуты для сложных компонентов (меню, модальные окна, карусели).
  - Alt-текст для всех изображений.
  - Пропуск к основному контенту (skip link).

### 9.5 Производительность

- Оценка Google PageSpeed Insights: не менее **85** баллов (mobile) и **92** баллов (desktop).
- Lazy loading для изображений и встроенного медиа.
- Сжатие изображений: WebP с fallback на JPEG.
- Минификация CSS, JS.
- Шрифты загружаются с `font-display: swap`.

### 9.6 Анимации и переходы

- Subtle-анимации появления карточек (fade-in + translateY).
- Плавные переходы между состояниями кнопок и элементов форм (CSS transitions).
- Не должны создавать проблем при `prefers-reduced-motion: reduce`.

## 12. Требования к безопасности

### 12.1 Аутентификация и авторизация

- Пароли хэшируются с использованием bcrypt (cost factor ≥ 12) или Argon2id.
- Все маршруты CMS защищены проверкой токена сессии.
- CSRF-токены во всех формах.
- JWT (если применяется): алгоритм RS256, срок действия ≤ 8 ч, хранение в httpOnly cookie.

### 12.2 Защита от инъекций

- Строки параметров запросов и поля форм — экранируются и валидируются на сервере.
- Использование параметризованных запросов к БД (ORM или prepared statements).
- HTML-контент из WYSIWYG — санируется (DOMPurify или аналог) перед сохранением.
- Загружаемые файлы — проверка MIME-типа на сервере (не только по расширению).

### 12.3 Сетевая безопасность

- Обязательный HTTPS (TLS 1.2+). Редирект с HTTP → HTTPS.
- HSTS-заголовок (Strict-Transport-Security).
- Content Security Policy (CSP): ограничение источников скриптов.
- X-Frame-Options: DENY (защита от clickjacking).
- X-Content-Type-Options: nosniff.
- Rate-limiting на форму входа: не более 5 попыток в час с одного IP.

### 12.4 Управление зависимостями

- Использовать только актуальные версии библиотек без известных уязвимостей (CVE).
- Регулярная проверка через `npm audit` / `pip-audit` или аналог.

### 12.5 Политика конфиденциальности

- Страница политики конфиденциальности обязательна.
- Форма обратной связи не собирает избыточных данных.
- Данные посетителей не передаются третьим сторонам, кроме аналитических систем с анонимизацией IP.

Портал считается принятым при:

1. Прохождении всех тестов из п. 18.1–18.6 без блокирующих ошибок.
2. Утверждении дизайна и пользовательского сценария заказчиком.
3. Наличии загруженного начального контента (п. 16.1).
4. Передаче документации (инструкций пользователя для администратора и модератора) и исходного кода.

| Низкая (запрос на улучшение) | 3 рабочих дня | По согласованию |

Разработчик проводит обучение для:

- Администраторов (4 часа): полное управление CMS, пользователи, настройки, журнал.

По завершении обучения предоставляются пошаговые инструкции (PDF + видеозаписи экрана).

### 19.4 Документация

Разработчик предоставляет:

- Инструкцию пользователя CMS (Администратор, Модератор) — на русском языке.
